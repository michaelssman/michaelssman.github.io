#### model

- DTO（Data Transfer Object）数据传输对象：在各个层各个服务之间传递数据。接口请求的参数
- PO（Persistent Object）持久化对象/domain：**数据库**实体类
- VO（View Object）视图对象：给前端，前端要啥就组装啥。pc、app、h5等前端传的查询参数个数不一样，pc可能多一点（例如：v1，v2，v3，v4，v5），app少一点（例如：v1，v2，v3）。后端controller根据不同的vo定义不同的接口，service用同一个即可（按照参数最多的定义）。
- BO（Business Object）业务对象：专门在service层干业务。比如下单BO，里面包含订单PO、用户PO、商品PO，还会算总价、优惠等逻辑。
- pojo是上面的老祖宗。
- DAO（数据访问对象）：接口/类，负责和数据库打交道，隔离业务逻辑和数据操作，比如OrderDAO里有saveOrder、getOrderById等。

如果把数据库的PO直接裸奔给前端，万一哪天改了表结构，前端就无法使用了。PO里如果有盐值、密码哈希，不能给前端。所以需要上面的，分层**解耦**，**安全**清晰。

上面的类型转换使用`BeanUtils.copyProperties`，这是运行时反射，性能差，字段名对不上时也不会报错。推荐使用`MapStruct`，在编译器帮你把get、set代码自动生成好了，零反射，高性能。

###### PageResponseResult

PageResponseResult继承ResponseResult。

```java
import lombok.Data;
import lombok.EqualsAndHashCode;

import java.io.Serializable;

@EqualsAndHashCode(callSuper = true)
@Data
public class PageResponseResult<T> extends ResponseResult<T> implements Serializable {
    private Integer currentPage;
    private Integer size;
    private Integer total;

    public PageResponseResult(Integer currentPage, Integer size, Integer total) {
        this.currentPage = currentPage;
        this.size = size;
        this.total = total;
    }
}
```
